# Design canon

Normative decisions for the project. Each entry is one addressable **`D-N`**
unit that the worker, triage, and validator agents read and cite (`per D-N`), and
that design sessions patch. This file holds **decisions and constraints only** —
never a description of how the code works, and never plain vocabulary (a canonical
name nobody argued about goes in `docs/vocabulary.md`, not here).

What earns a `D-N`, the entry format, and the authoring discipline (the "what did we
weigh?" admission test, the scope-bounding `Why:`, never renumbering) live in the
**`domain-modeling` skill** — the single place that teaches how to write canon, so
this file stays lean and this guidance can't drift from the skill that applies it.
